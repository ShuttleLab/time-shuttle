# BUILD_DONE.md — Time Shuttle Build Report

## What was built

Time Shuttle is a free, client-side Unix timestamp and date conversion tool. All conversions happen in the browser using native JavaScript `Date` and `Intl` APIs — zero backend.

### Features implemented
1. **Unix ↔ Date bidirectional conversion** — auto-detects 10-digit (seconds) vs 13-digit (milliseconds)
2. **Current timestamp live display** — updates every second, one-click copy
3. **Multi-format output** — Unix (s), Unix (ms), ISO 8601, RFC 2822, UTC string, localized date, relative time
4. **Timezone conversion** — uses `Intl.supportedValuesOf('timeZone')` for full IANA database, `Intl.DateTimeFormat` for conversion
5. **Relative time** — `Intl.RelativeTimeFormat` per locale (en: "3 days ago", zh: "3天前")
6. **Batch conversion** — multi-line textarea for bulk timestamp processing

### Pages built
- **Layer 1**: Homepage with tool-first layout (compact title → tool → features → how-it-works → why section)
- **Layer 3**: About (with FAQPage + HowTo schemas), Privacy, Terms (all bilingual)
- **Layer 4**: 4 SEO landing pages (en: 700-1500 words + 4 schemas each; zh: thin page mode)

### Architecture
- Next.js 16.2.6 + React 19.2.4
- `output: "export"` (static export) + Cloudflare Workers static assets
- next-intl URL-based i18n (`localePrefix: "as-needed"`, locales: `["en", "zh"]`)
- Tailwind CSS v4 + shadcn/ui (Base UI primitives)
- Geist + Geist Mono fonts
- sonner toast
- Palette picker (6 accent colors + light/dark/system)
- PWA (manifest + icons + service worker)

## Uncertain points

1. **shadcn UI components** — copied directly from base64-shuttle. The select component's `onValueChange` type required a wrapper `(v) => { if (v) setState(v); }` to handle nullable values. This works but is slightly different from base64-shuttle's pattern.
2. **Service worker** — the postbuild script generates `sw.js` with precache list. This is identical to base64-shuttle. PWA installability should work, but hasn't been tested in a real browser yet.
3. **No `next-env.d.ts`** — this file is auto-generated by Next.js on first build and is gitignored. It exists in the `.next/` directory.
